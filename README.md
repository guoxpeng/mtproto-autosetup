# 🚀 MTG 代理一键安装工具

> 你只需要 **复制→粘贴→回车**，3 步搞定一个 Telegram 代理！  
> 每次安装都会**自动生成一个全新的密码**，别人猜不到、撞不上。

---

## 📖 这是干什么的？

这个脚本可以在你的**云服务器（VPS）** 上搭一个**代理服务**。

搭好之后，你的 Telegram 就可以通过这个代理上网了 —— **不会被墙、不会被限速**。

> 💡 **不懂技术也没关系** —— 你只需要会复制粘贴就行！

---

## ✅ 开始前，先看看你的服务器行不行

| 需要什么 | 说明 |
|---|---|
| 🖥️ **一台云服务器** | 随便哪家都行，系统是 Ubuntu / Debian / CentOS |
| 🔐 **root 权限** | 安装的时候需要管理员身份（用 `sudo`） |
| 🌐 **能连上 GitHub** | 脚本要从 GitHub 下载程序 |
| 🔌 **开放端口** | 云服务商的控制台里要放行你选的端口（比如 443） |

> 🎯 **不用买域名！** 只需要填一个知名网站的地址（比如 `www.bing.com`）用来"伪装"就行。

---

## 🚀 一键安装（最简单的办法）

### 方法1：从 GitHub 直接装（推荐 🏆）

把下面这行命令**整个复制**到你服务器的终端里，回车就行：

```bash
sudo bash -c "$(curl -fsSL https://raw.githubusercontent.com/guoxpeng/mtproto-autosetup/main/mtg-manager.sh)" -- install
```

> 📌 **不用改任何东西**，直接复制粘贴就能用。  
> 脚本会自动选一个**空闲端口**（20000~60000 之间），不用担心被占用。  
> 安装完成后自动注册为全局命令 `mtg-manager`，之后直接用即可。  
> 即使 mtg 程序无法生成密钥，脚本也会**自动用本地随机方式生成**，保证一次成功。

### 还可以加这些设置

| 加什么 | 默认值 | 有什么用 |
|---|---|---|
| `--domain www.bing.com` | 自动用 `www.bing.com` | 伪装成这个网站 |
| `--port 8443` | **自动选一个空闲端口** | 手动指定代理端口 |
| `--version v2.2.8` | 自动用最新版 | 指定 mtg 的版本 |
| `--force` | 不强制 | 就算装过了也重新装 |
| `--help` | — | 查看所有可用命令 |

**举例：**

```bash
sudo bash mtg-manager.sh install --domain www.google.com --port 8443
```

## 🎮 装好之后怎么用？

### 常用命令（花 1 分钟看完就够了）

| 命令 | 有什么用 |
|---|---|
| `mtg-manager show` | 📋 **查看代理信息**（IP、端口、密码、二维码） |
| `mtg-manager key` | 🔑 **只看密码**（复制发给朋友） |
| `mtg-manager show --json` | 📊 用程序能读的格式显示信息 |
| `mtg-manager list-keys` | 📜 **查看所有历史密码**（以前用过的都在） |
| `mtg-manager show-key 0` | 🔍 查看第 0 号历史密码的详细信息 |
| `mtg-manager stats` | 📈 **看看现在有多少人在用** |
| `mtg-manager status` | ❓ 服务是不是在正常运行 |
| `mtg-manager logs` | 📝 看实时日志（出了啥问题） |
| `mtg-manager doctor` | 🏥 **全面体检**（10 项检查一键诊断） |
| `mtg-manager rotate-secret` | 🔄 **换一个新密码**（旧的立刻失效） |
| `mtg-manager update` | ⬆️ 升级到最新版本（配置和密码不变） |
| `mtg-manager uninstall` | 🗑️ 卸载（配置会保留，方便重装） |

> 💡 **第一次装好之后**，脚本会自动装成全局命令（支持管道和本地文件两种安装方式），以后直接打 `mtg-manager xxx` 就行，不用写路径。

> 🔄 **重启说明**：脚本通过 `systemctl enable mtg` 注册了开机自启，VPS 重启后代理会自动运行。防火墙规则也会自动持久化（iptables-persistent），重启后不会丢失。当前端口和密钥除非重新执行 `install --force`，否则即使重启也不会改变。

## 📱 在 Telegram 里怎么填？

装好之后执行 `mtg-manager show`，会看到这样的信息：

```
=============== MTProto 代理信息 ===============
  服务器地址 : 123.45.67.89     ← 复制这个
  端口       : 443              ← 复制这个
  Secret     : ee47ce5d...      ← 复制这个（完整复制，别漏）
  连接链接   : tg://proxy?...   ← 点这个自动填好
====================================================
```

### 手动填法

打开 Telegram → 设置 → 数据与存储 → 代理 → 添加代理：

| Telegram 里面的输入框 | 填什么 |
|---|---|
| 类型 | 选 **MTPROTO**（不要选 SOCKS5 或 HTTP） |
| 服务器地址 | 填上面看到的「服务器地址」 |
| 端口 | 填上面看到的「端口」 |
| 密钥 | 填上面看到的「Secret」，**完整复制，一个字都不能少** |

### 懒人填法

直接**点一下**那个 `tg://proxy?server=...` 的链接，Telegram 会自动帮你全部填好！

## 📜 密码管理（重要！）

### 每次安装都生成新密码

- `install`（安装）→ 生成一个**全新的、随机的**密码
- `rotate-secret`（轮换）→ 再生成一个**全新的**密码，旧的立刻作废
- 每次生成的密码都**不一样**，就跟每次换一把新锁一样

### 查看以前用过的密码

```bash
# 看所有历史密码
mtg-manager list-keys

# 看第 2 号密码的完整信息
mtg-manager show-key 2
```

> 🔒 **密码 = 你家门钥匙** —— 谁拿到密码谁就能用你的代理。  
> 不要发到群里、朋友圈。万一泄露了，执行 `rotate-secret` 换一把新锁。

## 🔧 出问题了怎么办？

按顺序试一试：

1. **`mtg-manager status`** — 看服务是不是在跑
2. **`mtg-manager doctor`** — 全面体检，看哪里亮了红灯
3. **`mtg-manager logs`** — 看实时日志，找报错信息
4. **`mtg-manager show`** — 如果 Secret 显示「(空)」，重新安装：加 `--force`
5. **去云服务商的控制台** — 看看安全组有没有放行端口
6. **换个网络试试** — 换手机流量 / 换 Wi-Fi，排除本地网络问题

## 🗂️ 文件都放在哪

```
/etc/mtg/
├── config.toml          ← 配置文件（不用管它）
├── secret               ← 当前密码
├── secret.latest        ← 当前密码的快捷方式
├── keys/                ← 所有历史密码都在这里
│   ├── secret-20260719-143022
│   └── secret-20260720-091815
├── backups/             ← 自动备份
│   ├── config.toml.20260719-143022
│   ├── secret.20260719-143022
│   └── keys.20260720-091815/
└── ...

/usr/local/bin/
├── mtg                  ← 代理程序本身
└── mtg-manager          ← 这个管理脚本
```

## 🧹 卸载

```bash
mtg-manager uninstall
```

- 会停止服务、删除程序
- **配置文件、历史密码、备份都会保留**（怕你后悔）
- 想彻底删干净：`sudo rm -rf /etc/mtg`

## ⚠️ 安全提醒

- 🔑 **密码不要发到群里或朋友圈** —— 谁拿到谁就能用
- 🔄 觉得不安全了就执行 `rotate-secret` 换密码
- 🌐 伪装域名选大网站（百度、必应、谷歌），不容易被识别
- 👨‍👩‍👧‍👦 一台服务器只给家人朋友用，别公开分享
- 🔒 密钥文件权限为 `600`（仅 root 可读），防止其他进程窃取
- 🧱 防火墙规则自动持久化，重启 VPS 后 iptables 规则不丢失
- 📋 请在法律允许的范围内使用
